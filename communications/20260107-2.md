### P0（必须改，否则实现会卡）——SC10 编辑确认流与后端 API/状态机不匹配

**你在 prototypes（产品原型）里明确有 SC10：Pre-check and Edit（允许用户修正 symbol/amount/value、处理歧义、确认 platform_guess、补 avg_price）**

* **没有任何 endpoint** 能让客户端把 SC10 的“手动编辑结果 / 平台确认 / avg_price 补充”提交回后端并触发“重新定价 + 重新生成 preview calc”。

1. **你的 SC10 设计实现不了**（或者会变成“编辑后还要重算一次报告”，但后端没有接口/状态）

  2. 返回 `status=needs_review`，携带 `ocr_assets`（以及 ambiguities）供 SC10 展示
* 用户在 SC10 完成编辑/确认后：

  * `POST /v1/upload-batches/{id}/confirm`（或扩展现有 `/resolve`）提交：

    * ambiguities resolutions
    * edits（symbol/amount/value/avg_price/pnl_percent/platform_guess override 等）
    * user timezone（如未有）
  * 后端再做 normalization + pricing + snapshot + preview calc
  * 然后才进入 SC11 Processing（Free）

**最小改动版本（推荐）**：扩展现有 `/resolve` 成 `/review`，把“resolve ambiguities”和“manual edits”合并成一个 payload，减少 endpoint 数量。

   * mobile env 加 `# iOS only / # Android only / # Optional`
   * backend env 把 Apple web auth 那组移到 optional 区
