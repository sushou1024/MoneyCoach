# Google Play Release Readiness (2026-02-12)

This document describes the Android/Google Play release configuration implemented in this repository.

## Scope of changes

1. Android release builds now require explicit upload-signing credentials instead of the debug keystore.
2. Android manifest permissions were reduced for Play policy compatibility.
3. Backend deploy wiring was aligned with actual CloudFormation parameters.
4. Google Play RTDN webhook audience (`GOOGLE_PUBSUB_AUDIENCE`) is now deployable via CloudFormation.
5. A manual GitHub Actions workflow was added to build a signed AAB and optionally upload it to Google Play.

## GitHub Actions configuration

### 1) Required Secrets

Set in: `Settings -> Secrets and variables -> Actions -> Secrets`

- `ANDROID_UPLOAD_KEYSTORE_BASE64`
- `ANDROID_UPLOAD_STORE_PASSWORD`
- `ANDROID_UPLOAD_KEY_ALIAS`
- `ANDROID_UPLOAD_KEY_PASSWORD`
- `GOOGLE_PLAY_SERVICE_ACCOUNT_JSON_BASE64`

Notes:
- `ANDROID_UPLOAD_KEYSTORE_BASE64` is the base64 content of your Android upload keystore (`.jks`/`.keystore`).
- `GOOGLE_PLAY_SERVICE_ACCOUNT_JSON_BASE64` is the base64 content of the Google service account JSON key with Android Publisher API access.

### 2) Required Variables

Set in: `Settings -> Secrets and variables -> Actions -> Variables`

Backend / billing:
- `GOOGLE_PLAY_PACKAGE_NAME`
- `GOOGLE_PLAY_PRODUCT_WEEKLY`
- `GOOGLE_PLAY_PRODUCT_ANNUAL`
- `GOOGLE_PUBSUB_AUDIENCE`

Mobile app runtime (release workflow validates these):
- `EXPO_PUBLIC_API_BASE_URL`
- `EXPO_PUBLIC_GOOGLE_IOS_CLIENT_ID`
- `EXPO_PUBLIC_GOOGLE_ANDROID_CLIENT_ID`
- `EXPO_PUBLIC_STRIPE_PUBLISHABLE_KEY`

Recommended additional runtime variables:
- `EXPO_PUBLIC_API_BASE_URL_ANDROID`
- `EXPO_PUBLIC_PWA_URL`
- `EXPO_PUBLIC_GOOGLE_ANDROID_CLIENT_ID_DEBUG`
- `EXPO_PUBLIC_GOOGLE_EXPO_CLIENT_ID`
- `EXPO_PUBLIC_GOOGLE_WEB_CLIENT_ID`
- `EXPO_PUBLIC_APPLE_IAP_PRODUCT_ID_WEEKLY`
- `EXPO_PUBLIC_APPLE_IAP_PRODUCT_ID_YEARLY`

## Workflows

### Backend deploy workflow

File: `.github/workflows/ci.yml`

Current Play-related parameter contract for backend deployment:
- `GooglePlayPackageName`
- `GooglePlayProductWeekly`
- `GooglePlayProductAnnual`
- `GooglePlayServiceAccountJsonBase64`
- `GooglePubSubAudience`

Obsolete variables were removed from the deploy path:
- `GOOGLE_PLAY_SUBSCRIPTION_ID`
- `GOOGLE_PLAY_BASE_PLAN_ID_WEEKLY`
- `GOOGLE_PLAY_BASE_PLAN_ID_YEARLY`

### Android release workflow

File: `.github/workflows/mobile-app-android-release.yml`

Trigger: `workflow_dispatch`

Inputs:
- `submit_to_play` (boolean)
- `track` (`internal|alpha|beta|production`)
- `release_status` (`draft|completed|inProgress|halted`)
- `version_code` (optional)
- `version_name` (optional)

Behavior:
1. Validates required Secrets/Variables.
2. Runs `expo prebuild` to generate `android/`.
3. Applies tracked Gradle patching for release signing/version overrides.
4. Decodes the upload keystore.
5. Builds a signed release AAB.
6. Uploads AAB as workflow artifact.
7. Optionally uploads to Google Play if `submit_to_play=true`.

## Android signing and versioning contract

Files:
- `mobile-app/scripts/prepare-android-release-gradle.mjs`
- `mobile-app/android/app/build.gradle` (generated by prebuild and patched in CI)

Release builds consume the following Gradle properties or environment variables:
- `MONEYCOACH_UPLOAD_STORE_FILE`
- `MONEYCOACH_UPLOAD_STORE_PASSWORD`
- `MONEYCOACH_UPLOAD_KEY_ALIAS`
- `MONEYCOACH_UPLOAD_KEY_PASSWORD`
- `MONEYCOACH_VERSION_CODE` (optional)
- `MONEYCOACH_VERSION_NAME` (optional)

If a release task is requested without signing credentials, the build fails explicitly.

## Permissions hardening

Files:
- `mobile-app/app.json`
- `mobile-app/plugins/withAndroidPlayPolicy.js`
- `mobile-app/android/app/src/main/AndroidManifest.xml`

The Android config now allows only required media + billing permissions and blocks legacy/high-risk permissions not used by the app's current features.
